# Skills Taxonomy & Matrix

This document defines the skills taxonomy and proficiency levels used by the matching engine to evaluate users against job descriptions.

## 1. Core Categories & Sub-Skills

### Data Engineering & Infrastructure
*   **Languages:** Python, SQL, Scala, Java, Go
*   **Databases (SQL/NoSQL):** PostgreSQL, MySQL, MongoDB, Redis, Cassandra, Neo4j
*   **Data Warehouses:** Snowflake, BigQuery, Redshift, ClickHouse
*   **ETL/ELT Tools:** Apache Airflow, dbt, Luigie, Prefect, Fivetran
*   **Big Data Frameworks:** Apache Spark, Hadoop, Flink, Kafka
*   **Cloud Platforms:** AWS (S3, EC2, EMR), GCP (GCS, Dataproc), Azure

### Software Engineering & IT Operations
*   **Backend Development:** Node.js, Django, FastAPI, Spring Boot
*   **Version Control & CI/CD:** Git, GitHub Actions, GitLab CI, Jenkins
*   **Infrastructure as Code:** Docker, Kubernetes, Terraform, Ansible
*   **Systems & Support:** Linux/Unix Administration, Networking Basics, ITIL

### Data Science & Analytics
*   **Analysis & Visualization:** Tableau, PowerBI, Looker, Excel, Pandas, NumPy
*   **Machine Learning:** Scikit-Learn, TensorFlow, PyTorch, MLOps

## 2. Proficiency Matrix
When assessing skill gaps, use the following 3-tier scoring system to evaluate both user skills and job requirements:

| Level | Label | Description | Expected Action for Gaps |
| :--- | :--- | :--- | :--- |
| **L1** | Foundational | Understands concepts; syntax familiar; needs guidance. | Tutorials, documentation, basic syntax projects. |
| **L2** | Intermediate | Can build independently; understands best practices. | Porting mini-projects, debugging, architectural patterns. |
| **L3** | Advanced | Production-level optimization; system design; mentoring. | Deep dives, performance tuning, system scaling. |
